# ЦЕХ — управление швейным цехом

Веб-приложение (PWA) для небольшого/среднего швейного производства: партии,
сотрудники, этапы производства, задачи, зарплаты, авансы, расходы и
себестоимость. Mobile-first, работает как приложение с телефона.

**Стек:** vanilla JS SPA (без сборки) · Supabase PostgreSQL · деплой на Vercel.

Главное: вся бизнес-логика и контроль доступа — в RPC-функциях PostgreSQL
(`SECURITY DEFINER`). Клиент не имеет прямого доступа к таблицам. Это гарантирует
корректность расчётов зарплаты/себестоимости и защиту от обхода прав.

Подробности архитектуры: [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md).

---

## Возможности

- Вход по **телефону + PIN** (PIN хранится как bcrypt-хэш, блокировка после 5 ошибок).
- Роли: admin, technologist, cutter, seamstress, ironer, packer, dismissed.
- Производственный конвейер: раскрой → пошив → утюжка → упаковка → завершено.
- Назначение задач со ставкой, отметка выполнения (накопительно, без превышения плана).
- Зарплата считается **только из `work_records`**; защита от двойного начисления.
- Выплаты (зарплата/аванс/бонус/штраф), запросы аванса с подтверждением.
- Расходы цеха и по партии, расчёт себестоимости, прибыли и маржи.
- Уведомления внутри приложения, мультиязычность (ru/ky/en), PWA.

## Установка

### 1. Создать проект Supabase

1. Зарегистрируйтесь на [supabase.com](https://supabase.com), создайте проект.
2. Откройте **SQL Editor** и выполните по очереди три файла из папки `supabase/`:
   1. `01_schema.sql` — таблицы, типы, индексы, RLS.
   2. `02_functions.sql` — все RPC-функции и гранты.
   3. `03_seed.sql` — демо-данные (сотрудники, партии, задачи).
3. В **Project Settings → API** скопируйте `Project URL` и `anon public` ключ.

### 2. Подключить фронтенд

Откройте [js/config.js](js/config.js) и впишите свои значения:

```js
export const SUPABASE_URL = 'https://ВАШ-ПРОЕКТ.supabase.co';
export const SUPABASE_ANON_KEY = 'ВАШ-ANON-КЛЮЧ';
```

> Сборки нет, поэтому переменные окружения не подставляются автоматически.
> Альтернатива редактированию `config.js` — создать `js/env.js` с
> `window.__ENV__ = { SUPABASE_URL, SUPABASE_ANON_KEY }` и подключить его в
> `index.html` перед `js/app.js`. См. `.env.example`.

### 3. Запустить локально

Нужен любой статический сервер (из-за ES-модулей нельзя открывать `file://`):

```bash
npx serve .
# или
python -m http.server 8000
```

Откройте `http://localhost:3000` (или указанный порт).

## Обновление уже работающей БД

Если вы уже выполнили `01–03` ранее, для перехода на валютную модель ткани
(USD + курс) и новые права выполните в SQL Editor по порядку:

1. `supabase/05_update.sql` — меняет структуру таблицы `batches`.
2. `supabase/02_functions.sql` — заново (обновляет логику и гранты).
3. `supabase/03_seed.sql` — по желанию, обновит демо-данные.

## Модель ткани и валюты

- Ткань закупается по **метру** или по **килограмму**, цена — в **долларах**.
- У каждой партии свой **курс доллара** (сом за $1).
- Стоимость ткани = кол-во × цена($) × курс → в сомах. Себестоимость
  (ткань + расходы + работа) считается в сомах.
- **Закройщик** может создать партию (без денежных полей) и указать
  метраж/вес и фактическое количество после раскроя. Цену ткани, курс и
  цену продажи задаёт только владелец (admin).
- **Технолог** выдаёт зарплаты и видит начисления сотрудников, но НЕ видит
  расходы, цену ткани и себестоимость партий.

## Деплой на Vercel

1. Запушьте репозиторий на GitHub.
2. В Vercel: **Add New → Project**, импортируйте репозиторий.
   Framework Preset — **Other** (сборка не нужна, Output Directory — корень).
3. Убедитесь, что `js/config.js` содержит ваши Supabase URL и ключ
   (они попадут в статический бандл — это нормально, anon-ключ публичный,
   доступ всё равно ограничен токеном сессии внутри RPC).
4. Deploy.

[vercel.json](vercel.json) уже настроен:
- SPA-rewrites на `index.html`;
- `index.html` / `sw.js` — `no-cache` (обновления не ломаются старым кэшем);
- `css/`, `icons/` — `immutable` (версионируются через `?v=`);
- `js/` — `must-revalidate` (модули не версионируются пофайлово).

> При изменении CSS поднимите версию в ссылке `styles.css?v=...`
> (в `index.html` и `sw.js`) и значение `APP_VERSION` в `sw.js`.

## Установка на телефон (PWA)

Приложение можно установить на телефон как обычное приложение:

- **Android (Chrome):** откройте сайт → раздел «Профиль» → кнопка
  «Установить на телефон». Или меню браузера (⋮) → «Установить приложение».
- **iPhone (Safari):** откройте сайт → «Поделиться» (↑) → «На экран „Домой"».

Иконки лежат в [icons/](icons/): `icon.svg` (вектор) и PNG `icon-192/512/180`.
Пересоздать PNG из дизайна можно командой `node scripts/make-icons.mjs`.

## Ключевые бизнес-правила (кратко)

- `total_amount = quantity × rate_per_unit` на каждую запись `work_records`.
- `Остаток = Начислено + Бонусы − Штрафы − Выплачено`.
- Нельзя сдать больше плана; нельзя `quantity ≤ 0`; повторный клик блокируется.
- Сотрудников и партии нельзя удалять физически — только `dismissed` / `archived` / `cancelled`.
- Перевод этапа требует завершения задач; принудительный перевод — только admin с подтверждением.
- Себестоимость = ткань + расходы по партии + работа по партии.

Полная матрица прав и правила — в [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md).
